C4Image Class Reference

Inherits from C4Control : UIControl
Declared in C4Image.h

Overview

This document describes the C4Image class. A C4Image object provides access for creating, showing, interacting and manipulating images. C4Image is a subclass of C4Control and so inherits its animation, gesture and notification abilities.

C4Image also takes advantage of its underlying C4Layer to allow for filtering the content of an image. You can apply filters that affect only the image (e.g. hueAdjust, vibranceAdjust, etc.) or use another image as the background for a particular blend or composite filter (e.g. additionComposite:, overlayBlend:, etc).

When working with blends and composite methods, it is recommended that the images you use are of the same size.

Composite Operations

  • – additionComposite:

    Addition composite filter
  • – colorBlend:

    Color blend filter Uses the luminance values of the background with the hue and saturation values of the source image.
  • – colorBurn:

    Color burn filter Darkens the background image samples to reflect the source image samples.
  • – colorDodge:

    Color dodge filter.
  • – darkenBlend:

    Darken blend filter Creates composite image samples by choosing the darker samples (from either the source image or the background).
  • – differenceBlend:

    Difference blend filter Subtracts either the source image sample color from the background image sample color, or the reverse, depending on which sample has the greater brightness value.
  • – exclusionBlend:

    Exclusion blend filter Produces an effect similar to that produced by the Difference Blend filter but with lower contrast.
  • – hardLightBlend:

    Hard light blend filter Either multiplies or screens colors, depending on the source image sample color.
  • – hueBlend:

    Hue blend filter Uses the luminance and saturation values of the background with the hue of the source image.
  • – lightenBlend:

    Lighten blend filter Creates composite image samples by choosing the lighter samples (either from the source image or the background).
  • – luminosityBlend:

    Luminosity blend filter Uses the hue and saturation of the background with the luminance of the source image.
  • – maximumComposite:

    Maximum compositing filter Computes the maximum value, by color component, of two input images and creates an output image using the maximum values.
  • – minimumComposite:

    Minimum compositing filter Computes the minimum value, by color component, of two input images and creates an output image using the minimum values.
  • – multiplyBlend:

    Multiply blend filter Multiplies the source image samples with the background image samples.
  • – multiplyComposite:

    Multiply compositing filter Multiplies the color component of two input images and creates an output image using the multiplied values.
  • – overlayBlend:

    Overlay blend filter Either multiplies or screens the source image samples with the background image samples, depending on the background color.
  • – saturationBlend:

    Saturation blend filter Uses the luminance and hue values of the background with the saturation of the source image.
  • – screenBlend:

    Screen blend filter Multiplies the inverse of the source image samples with the inverse of the background image samples.
  • – softLightBlend:

    Soft light blend filter Either darkens or lightens colors, depending on the source image sample color.
  • – sourceAtopComposite:

    Source-atop compositing filter Places the source image over the background image, then uses the luminance of the background image to determine what to show.
  • – sourceInComposite:

    Source-in compositing filter Uses the background image to define what to leave in the source image, effectively cropping the image.
  • – sourceOutComposite:

    Source-out compositing filter Uses the background image to define what to take out of the first image.
  • – sourceOverComposite:

    Source-over compositing filter Places the background image over the source image.
